01 · Product thesis

From cockpit to operating system.

Updog is not only where decisions are modeled; it is where they are operationalized across vehicles. The interface must reduce switching costs, expose risk early, and preserve trust under LP scrutiny.

North star: every high-value surface answers four questions in the first viewport: What is true now? What changed? What is due? What can I do next?
A

Live truth with provenance.

Metrics carry source, sync time, assumption version, and editor attribution.

B

Cross-entity awareness.

Deadlines, cash events, blockers, and context surface across all active funds/SPVs by default.

C

Execution continuity.

Every material scenario can produce assigned actions, due dates, and shareable outputs.

02 · Principles

Twelve rules for every surface.

P / 01

Numbers before chrome.

Decision-grade figures outrank navigation and decorative controls.

P / 02

One truth-line.

Canonical metrics keep one order and grammar across contexts.

P / 03

Density is respect.

Compact expert views are the default; whitespace organizes, not dilutes.

P / 04

Multi-entity by design.

Fund, SPV, sidecar, portfolio company, and LP perspectives are first-class.

P / 05

Assumptions are objects.

Material assumptions are versioned, diffable, attributable, and timestamped.

P / 06

Urgency is explicit.

Hard deadlines, cash impact, and blockers are visible without drill-down.

P / 07

Keyboard is default.

Command, search, saved views, assignment, and share flows remain fast and reachable.

P / 08

Motion explains state.

Animations express causality and continuity, never decoration.

P / 09

Collaboration resolves into ownership.

Mentions, comments, and attachments create tasks, decisions, or audit notes.

P / 10

Solo-safe by default.

Every collaborative workflow must still function for one operator.

P / 11

Investor-readiness built in.

LP-safe snapshots are immutable, scoped, freshness-labeled, and assumption-labeled.

P / 12

Context never disappears.

Overlays preserve place, scroll position, filters, and selected entity.

07 · Interaction model

Action stays near the evidence.

Pattern

Cards open work panels.

Project, task, company, SPV, assumption, and snapshot cards open context-preserving modals or drawers.

Pattern

Charts reveal proof.

Hover and focus states expose values, drivers, deltas, source timestamps, and relevant assumptions.

Pattern

Comments produce ownership.

Mentions, replies, attachments, and decisions resolve into tasks, owners, due dates, or audit notes.

Pattern

Primary actions are black.

Create, Invite, New Project, Save, Approve, and Share use the same high-contrast button language.

Pattern

Local actions stay local.

Overflow menus belong to the card, metric, row, or chart region they affect.

Pattern

Saved views are portable.

Filters, entity scope, time range, and visible columns can be saved, shared, and reopened by command.

08 · Context-preserving work panels

The dashboard should never fall away.

Task · Assumption · SPV · Snapshot

Open material objects in drawers or modals unless the user is changing workspace. Background context remains visible but non-interactive.

AssigneeTagsStatusDue dateSave
Behavior

Dim, blur, return.

Panels fade/scale in over the same dashboard and close back to the exact prior viewport, selected entity, scroll position, and filters.

Behavior

Editable metadata first.

Assignee, tags, status, due date, priority, and linked entity remain editable inline before description or comments.

Behavior

Tabs preserve state.

Description, Comment, Settings, History, and Audit tabs swap content without closing the panel or resetting form state.

Behavior

Cancel is safe; Save is explicit.

Material edits never imply side effects. Save, approve, publish, or share must be visually explicit and auditable.

09 · Motion grammar

Motion should show continuity.

Loading

Keep chrome stable.

Sidebar and header remain visible. Main content reserves space with skeletons before fading in.

Overlay

Dim, blur, return.

Modal overlay fades in 150–200ms; panel scales from 96% to 100% in 180–240ms, then returns to the same context.

Hover

Reveal, don’t decorate.

Tooltips reveal values, sources, owners, deltas, and drivers. Hover regions should be forgiving.

Accessibility

Respect reduced motion.

All transitions degrade to instant state changes when reduced-motion is enabled.

10 · Collaboration model

Comments are not social feed.

Collaboration rule: comments are never free-floating. Each thread belongs to a task, scenario, company, assumption, LP snapshot, SPV, document, cash event, or decision. Mentions create assignable work. Attachments inherit context. Resolved threads remain visible in audit history.
Mention

@ creates responsibility.

Mentions should support autocomplete, token styling, notification routing, and optional task creation.

Attachment

Files inherit context.

Uploaded files carry object, entity, author, timestamp, and visibility scope.

Resolve

Decisions leave a trace.

Resolved comments should preserve who resolved, when, why, and what object changed.

11 · Distinctiveness guardrails

Different, but never fragile.

Distinctive by utility

Own the decision-to-operations loop.

Insight should always end in concrete execution paths.

Not distinctive by noise

Avoid novelty without leverage.

No decorative motion, social-feed clutter, or motifs that lower analytical legibility.

Trust guardrail

No unsupported claims for material numbers.

Generated recommendations must cite drivers, assumptions, source data, and confidence.

Control guardrail

No hidden financial side effects.

Material edits require explicit confirmation and visible audit trace.

13 · Acceptance rubric

A surface is ready when it passes all checks.

Truth first. Dominant element is a decision-grade metric, variance, or explicit question.
Live + accountable context. Freshness, owner, assumption version, and last material change are visible.
Multi-entity clarity. User can move from aggregate view to specific fund/SPV state in two interactions or fewer.
Screen grammar holds. Persistent navigation, command header, decision canvas, and context rail are all legible.
Operational urgency visible. Hard deadlines, blockers, and estimated financial impact are legible without hunting.
Drill-down preserves place. Modals/drawers return user to the same viewport, filters, and selected entity.
Keyboard parity. Core flows work via command/search/tab with visible focus states.
Collaboration resolves into ownership. Mentions, comments, and attachments create tasks, decisions, or audit records.
Motion is purposeful. Animations explain state, preserve continuity, and honor reduced-motion settings.
LP-safe sharing. Shared views are scoped, timestamped, immutable, freshness-labeled, and include change summary.
