`shared/schema-lp-sprint3.ts` against the live Neon endpoint so that no

presence, then applies per-cluster verdicts from the research matrix:

- **11 hard-deletes** (cohort cluster + notion cluster + portfolio-optimization
  cluster) — zero live-DB backing, zero mounted code paths OR dead service code.
  Delete the table exports, delete the dependent routes/services, unmount the
  one router that is still wired, delete the 4 tests that directly import the
  deleted Drizzle symbols, delete the orphaned `shared/migrations/*.sql` files
  that live outside the canonical `migrations/` directory, and strip the
  orphaned type re-exports in `shared/types/database.ts`.
- **15 LP tables** — introspect-first. If the introspect output shows the LP
  tables ARE in live Neon (because `migrations/001_lp_reporting_schema.sql`,
  `002_lp_reporting_indexes.sql`, `004_lp_sprint3_tables.sql` were applied at
  some point), flip the verdict to `keep-as-is` even though the routes are
  unmounted (the tables carry data). If they are NOT in live Neon, delete the
  schema files entirely since the routes are not mounted.
- **1 `jobOutbox` table** — keep-add-migration. Confirm via introspect whether
  the table exists. If not, re-emit
  `shared/migrations/0001_create_job_outbox.sql` into the canonical
  `migrations/` path so it gets applied on the next deploy. (`jobOutbox` has a
  live consumer at `server/services/variance-alert-automation.ts`, so deletion
  is not an option.)
- **17 other introspect-first tables** (pipeline / scoring / market-research /
  custom-fields / snapshot-comparisons / variance-planner / fund-strategy /
  portfolio-scenarios / reserve-allocation-strategies / performance-forecasts /
  monte-carlo-simulations / reallocation-audit) — introspect, and if any are
  absent from live Neon AND have zero active code references, flag for the human
  reviewer before deleting. Default verdict: `keep-as-is` unless introspect
  proves absent.
- **22 keep-as-is tables** — active surface, no action.
