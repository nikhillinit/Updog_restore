<objective>
Fix the bytes-as-milliseconds bug in `MonteCarloPerformanceTracker.trackMemoryUsage` that
causes `npm run phoenix:truth` to print false-positive critical alerts like
`Performance Alert: memory_simulation_complete took 36110552ms (critical)` every time
the Monte Carlo engine runs to completion.

Purpose: REQ-TEST-02. The slow-test threshold warning that pollutes
phoenix:truth output comes from production code, not a test file.
`trackMemoryUsage` passes `memoryUsage.heapUsed` (bytes, always tens of
millions) into the `duration: number` slot of `monitor.track()`. The operation
name `memory_simulation_complete` is not in the threshold map, so it falls back
to `api_request` thresholds (critical = 5000ms) and fires a critical alert on
every simulation. The truth-case suite at
`tests/unit/truth-cases/backtesting-scenario.test.ts` runs the real Monte Carlo
engine unmocked, so the alert fires on every phoenix:truth invocation.

Fix approach (Option A.i from planner investigation): rewrite the internal
`monitor.track(...)` call inside `trackMemoryUsage` to pass `0` as the duration
and put the heap values in the metadata object. No public API change, no caller
change, no downstream consumer is affected (audit shows zero code reads
`memory_*` metric durations — only `metric_recorded` and `performance_alert`
events and `monitor.exportMetrics()` are observed, and they all receive the
correct shape).

Output: a fixed `performance-monitor.ts` file with the bytes-as-duration bug
gone, a new `tests/unit/middleware/performance-monitor.test.ts` regression test
that locks in the fix, and a clean `npm run phoenix:truth` run with zero
critical-severity lines for memory metrics. </objective>

Replace it with:

```typescript
  /**
   * Record memory usage as a gauge metric.
   *
   * NOTE: `PerformanceMonitor.track` takes a `duration: number` in milliseconds.
   * Memory usage is not a duration — it is a gauge. We pass `0` for duration so
   * the threshold check in `track()` never marks this metric as slow/critical
   * (which would otherwise fire bogus `performance_alert` events every simulation,
   * polluting `npm run phoenix:truth` output). Heap values are carried in metadata
   * where any real consumer can read them.
   *
   * See REQ-TEST-02 in `.planning/phases/05-test-hygiene-resurrection/`.
   */
  trackMemoryUsage(operation: string, memoryUsage: NodeJS.MemoryUsage) {
    monitor.track(`memory_${operation}`, 0, 'computation', {
      heapTotal: memoryUsage.heapTotal,
      heapUsed: memoryUsage.heapUsed,
      external: memoryUsage.external,
      rss: memoryUsage.rss,
    });
  }
```
